Overview

DeepCatch M is a production-grade MLOps pipeline for MRI segmentation, designed for multi-organ analysis including Subcutaneous Fat (SF) and Abdominal Visceral Fat (AVF). The project addresses the full ML lifecycle from data versioning to containerized deployment, with strict reproducibility for clinical applications.

PyTorch MLFlow DVC Medical Imaging Segmentation

Multi-organ MRI Segmentation Models

Problem

Manual segmentation of organs in MRI scans is time-consuming and subject to inter-observer variability. Need to compare multiple SOTA architectures for optimal performance on medical imaging tasks.

Approach

Implemented and evaluated multiple segmentation architectures (U-Net, U-Net++, nnU-Net) through a unified model factory pattern, enabling systematic comparison and architecture search.

Model Architectures

  • U-Net: Baseline encoder-decoder with skip connections (~7.8M params)
  • U-Net++: Dense nested skip pathways for better gradient flow (~9.2M params)
  • nnU-Net: SOTA with residual blocks, instance norm, deep supervision (~31M params)

Technical Details

  • Multi-class Dice loss with deep supervision for auxiliary losses
  • Multi-GPU training with DataParallel and mixed precision (FP16)
  • Optimized data loading with caching and prefetching

Target Structures

  • Subcutaneous Fat (SF)
  • Abdominal Visceral Fat (AVF)

MLOps Pipeline Template

Problem

Medical imaging ML requires strict reproducibility, audit trails for compliance, and systematic model comparison. Ad-hoc workflows led to difficulty reproducing results and deploying models.

Approach

Built an end-to-end MLOps pipeline template with hybrid architecture: native GPU training for performance, containerized serving for portability and reliability.

Pipeline Components

  • Data Versioning (DVC): Git-like versioning for large medical datasets with deduplication
  • Experiment Tracking (MLFlow + W&B): Metrics, parameters, artifacts, and prediction visualization
  • Hyperparameter Optimization (Ray Tune): ASHA scheduler with multi-GPU parallel trials
  • Model Serving (FastAPI + Docker): REST API with caching, health checks, and monitoring
  • Deployment (HuggingFace Hub): Automated model publishing with model cards
