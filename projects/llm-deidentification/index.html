Overview

Medical data processing requires strict compliance with privacy regulations (HIPAA, GDPR) while enabling research and international collaboration. This project develops AI-powered systems for automated de-identification of Protected Health Information (PHI) and multilingual translation of medical documents with quality assurance.

Multi-modal PHI De-Identification Pipeline

Problem

Medical images and documents contain Protected Health Information (PHI) embedded in multiple modalities: burned-in text on images, DICOM metadata, PDF documents, and handwritten annotations. Manual de-identification is time-consuming and error-prone.

Approach

Developed a comprehensive multi-modal pipeline that combines OCR, Named Entity Recognition, and Large Language Models to detect and anonymize PHI across different data types.

Technical Pipeline

  • Image PHI Detection: OCR extracts text from images; NER identifies PHI entities for pixel-level masking
  • Metadata Anonymization: LLM-based parsing and anonymization of DICOM headers and document metadata
  • Document Processing: PDF text extraction with layout-aware PHI detection
  • Quality Assurance: Multi-stage validation to ensure no PHI leakage

Multilingual Translation System

Problem

International regulatory submissions and medical documentation require accurate translation into multiple languages. Medical terminology must be consistent across documents, and LLM-based translation can suffer from hallucinations that introduce dangerous inaccuracies.

Approach

Engineered a scalable multilingual translation pipeline from Korean to 5 target languages, integrating Translation Memory for consistency and cost reduction, with statistical quality checks to detect and correct hallucinations.

Technical Architecture

  • Translation Memory (TM): Persistent database of approved translations; reduces API costs and ensures terminology consistency
  • LLM Translation: Uses large language models for context-aware translation of new segments
  • Consistency Checker: Statistical analysis to validate terminology usage across documents
  • Hallucination Detection: Automated checks for numerical consistency, entity preservation, and semantic alignment
  • Auto-Correction: Rule-based correction for detected issues before human review

Target Languages

  • Korean (source) to English, Japanese, Chinese, Spanish, German
