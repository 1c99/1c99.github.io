Overview

Medical data processing requires strict compliance with privacy regulations (HIPAA, GDPR) while enabling research and international collaboration. This project develops AI-powered systems for automated de-identification of Protected Health Information (PHI) and multilingual translation of medical documents with quality assurance.

Multi-modal PHI De-Identification Pipeline

Problem

Medical images and documents contain Protected Health Information (PHI) embedded in multiple modalities: burned-in text on images, DICOM metadata, PDF documents, and handwritten annotations. Manual de-identification is time-consuming and error-prone.

Approach

Developed a comprehensive multi-modal pipeline that combines OCR, Named Entity Recognition, and Large Language Models to detect and anonymize PHI across different data types.

Two-Pronged Approach

  • Metadata Anonymization: LLM parses DICOM headers to identify PHI fields (Patient Name, ID, Birth Date) and generates consistent anonymized mappings while preserving non-PHI metadata (Modality, dimensions)
  • Burn-in Text Detection: OCR extracts all text regions with bounding boxes; LLM/NER classifies each as PHI or non-PHI; pixel-level black mask applied over PHI regions while preserving markers (L/R, PA)

Multilingual Translation System

Problem

International regulatory submissions and medical documentation require accurate translation into multiple languages. Medical terminology must be consistent across documents, and LLM-based translation can suffer from hallucinations that introduce dangerous inaccuracies.

Approach

Engineered a scalable multilingual translation pipeline from Korean to 5 target languages, integrating Translation Memory for consistency and cost reduction, with statistical quality checks to detect and correct hallucinations.

Three Translation Approaches

  • Approach 1 - Distributed Ollama + Ray: Load-balanced distribution across multiple Ollama endpoints with async I/O for high throughput
  • Approach 2 - Multi-Agent Workflow: 4 specialized AI agents (Translator → Reviewer → Fixer → Supervisor) mimicking professional translation teams with glossary enforcement
  • Approach 3 - M2M100 + Caching: Facebook's M2M100 model with SQLite translation cache and HTML-aware processing

Quality Assurance Features

  • Translation Memory (TM): SQLite-based persistent cache reduces API costs and ensures terminology consistency
  • Glossary Support: Domain-specific medical terminology enforcement (e.g., "aorta" → "大動脈")
  • Hallucination Detection: Statistical checks for numerical consistency, entity preservation, and semantic alignment
  • HTML Preservation: Markup tags preserved during translation of structured content

Target Languages

  • Korean (source) to English, Japanese, Chinese, Spanish, German (+ Arabic RTL support)
