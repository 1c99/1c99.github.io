Overview

DeepCatch X is an AI-powered medical imaging platform that extracts 3D aortic structures from 2D chest X-rays, enabling non-invasive screening for thoracic aortic diseases. The project encompasses end-to-end development from research prototypes to production-ready deployment, including novel deep learning architectures, distributed inference systems, and regulatory compliance pipelines.

Conditional Diffusion Model for X-ray-to-Aorta Extraction

Problem

Traditional GAN-based approaches for medical image synthesis suffered from mode collapse and produced artifacts that could lead to clinical misinterpretation. We needed a more reliable generative model that could faithfully reconstruct 3D aortic structures from 2D projections.

Approach

Pioneered a Conditional Diffusion model architecture specifically designed for X-ray-to-Aorta extraction. The model conditions on the input X-ray and iteratively denoises to produce high-fidelity aortic segmentations. Additionally proposed LDM-based volumetric reconstruction for full 3D geometry recovery.

Architecture diagram coming soon

Technical Details

  • Implemented custom conditioning mechanism for X-ray feature injection
  • Accelerated training via Distributed Data Parallel (DDP) across multiple GPUs
  • Used Gradient Accumulation to simulate large-batch learning on memory-constrained hardware
  • Proposed Latent Diffusion Model (LDM) variant for volumetric reconstruction

Results

Superior Fidelity vs GANs
Reduced Synthesis Artifacts

Ray Distributed Inference Pipeline

Problem

Production deployment required processing thousands of X-ray studies efficiently across heterogeneous hardware. Single-node inference created bottlenecks that prevented scaling to clinical workloads.

Approach

Engineered a scalable inference pipeline using Ray Distributed computing framework. Designed adaptive DAG scheduling to optimize task allocation across multi-node workstations with varying GPU capabilities.

Technical Implementation

  • Implemented Ray actors for stateful model serving
  • Designed adaptive DAG scheduling based on hardware availability
  • Applied hardware-aware optimizations: TensorRT compilation, ONNX export, FP16 precision
  • Built auto-scaling mechanisms for dynamic workload handling

Results

6x Batch Throughput
2.8x Faster Per-Case Inference

Nuitka Compilation Pipeline

Problem

On-premise deployment at hospital sites required complex Python environment setup, creating significant installation overhead and potential dependency conflicts.

Approach

Built a Nuitka compilation pipeline to deploy deep learning models as standalone executables. This eliminated the need for Python installation on target machines and simplified the deployment process.

Technical Details

  • Refactored codebase for compilation compatibility
  • Configured Nuitka for PyTorch and CUDA library bundling
  • Created automated build scripts for cross-platform deployment
  • Implemented license protection mechanisms

Results

73% Codebase Reduction
Simplified On-Premise Setup

BioAge Prediction Module

Problem

Research-grade biological age prediction algorithms were implemented in R, making integration with the production C++ pipeline challenging and introducing performance overhead.

Approach

Ported the research-grade R algorithms to a C++ production environment, ensuring numerical equivalence while achieving significant performance improvements.

Technical Details

  • Analyzed and documented the R algorithm implementation
  • Implemented equivalent C++ code with Eigen for linear algebra
  • Validated numerical accuracy against R reference implementation
  • Integrated with existing C++ inference pipeline
